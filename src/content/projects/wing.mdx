### 프로젝트 개요

복잡하게 얽혀 있는 국제 정세와 투자 관련 뉴스를 사용자가 쉽고 빠르게 파악할 수 있도록 돕는 시각화 서비스입니다. AI가 분석한 뉴스 데이터 간의 연관성을 가중치 및 감성 점수 기반의 그래프 인터페이스로 구현하여 직관적인 정보 탐색 경험을 제공합니다. 사용자는 노드와 엣지로 연결된 시각화 화면을 통해 뉴스 간의 유기적인 관계를 파악하고, 투자 결정에 필요한 핵심 인사이트를 효율적으로 도출할 수 있습니다.

### 주요 기능

- Force-Directed Graph 시각화: 뉴스 키워드(Node)와 연관 관계(Link)를 물리 엔진 기반의 그래프로 표현
- 인터랙티브 조작: D3 Zoom/Pan 및 Drag & Drop 지원으로 자유로운 탐색 가능
- 동적 스타일링: 데이터 가중치(weight)와 감성 분석 결과(sentiment)에 따른 노드 크기 및 링크 색상/두께 변화
- 반응형 렌더링: 투자 모드/일반 모드 전환 및 링크 선택 시 즉각적인 하이라이팅 처리

### 문제점 & 해결 방법

    1. **D3 연산 비용과 React 렌더링 사이클 충돌 문제**
        - **문제**: D3의 `forceSimulation`은 초기 연산 비용이 높습니다. React의 상태가 변할 때마다(예: 링크 선택) 컴포넌트가 리렌더링되면서 그래프 시뮬레이션이 처음부터 다시 실행되어 화면이 깜빡이고 성능이 저하되는 문제가 있었습니다.
        - **해결**: 렌더링 로직을 두 개로 분리하여 역할을 명확히 했습니다.
            1. 시뮬레이션 Effect: 데이터(`graphData`)가 변경될 때만 무거운 `forceSimulation`을 실행하여 좌표를 계산합니다.
            2. 스타일링 Effect: 단순 상태 변경(`selectedLink`, `isInvestmentMode`) 시에는 시뮬레이션을 재실행하지 않고, D3의 DOM 조작만으로 색상과 투명도(`transition`)만 부드럽게 업데이트했습니다.
    2. **빈번한 상태 변화로 인한 불필요한 리렌더링**
        - **문제:** 사용자가 그래프를 클릭하거나 모드를 변경할 때마다 전역 상태가 바뀌는데, 이와 무관한 컴포넌트까지 영향을 받거나 불필요한 연산이 발생할 우려가 있었습니다.
        - **해결:** Zustand Selector 패턴을 적용해 `useGraphStore((state) => state.specificValue)` 형태로 필요한 상태만 구독(Subscribe)하도록 설계했습니다. 이를 통해 스토어 내의 다른 값이 변하더라도, 해당 컴포넌트는 정확히 의존하는 데이터가 변할 때만 반응하도록 렌더링 범위를 최적화했습니다.
    3. **다양한 데이터 스케일의 시각적 표현**
        - **문제:** 노드와 링크의 가중치(weight) 편차(최소값 vs 최대값)가 커서, 단순히 값을 픽셀로 변환하면 너무 작거나 큰 요소가 생겨 시인성이 떨어졌습니다.
        - **해결:** 데이터의 최소/최대값을 구해 0 ~ 1 사이로 정규화(Min-Max 정규화)한 뒤, 이를 기반으로 적절한 픽셀 범위(노드: 15px ~ 45px, 링크 두께: 2px ~ 6px)로 매핑하는 로직을 구현하여 데이터 왜곡 없이 시각적 균형을 맞췄습니다.
